c#中的函数传递参数是值传递还是

2023/12/8 10:36:11

在 C# 中，函数参数的传递方式取决于参数的类型。对于值类型（例如整数、浮点数、结构体等），参数是按值传递的，这意味着在函数调用时会复制参数的值传递给函数内部。

而对于引用类型（例如类实例、数组、接口等），参数是按引用传递的，这意味着函数内部使用的是参数的引用，因此在函数内部对参数进行的修改会影响到原始参数。

下面是一个示例来说明这一点：

csharpCopy Code
using System;

class Program
{
    static void Main()
    {
        int x = 10;
        MyClass obj = new MyClass();

        // 传递值类型参数
        UpdateValue(x);
        Console.WriteLine("x = " + x); // 输出 10，因为参数是按值传递的

        // 传递引用类型参数
        UpdateReference(obj);
        Console.WriteLine("obj.Value = " + obj.Value); // 输出 20，因为参数是按引用传递的
    }

    static void UpdateValue(int num)
    {
        num = 20; // 修改局部变量的值
    }

    static void UpdateReference(MyClass obj)
    {
        obj.Value = 20; // 修改引用类型对象的成员
    }
}

class MyClass
{
    public int Value { get; set; } = 10;
}

在上述示例中，UpdateValue 函数和 UpdateReference 函数演示了值类型和引用类型参数的传递行为。UpdateValue 函数修改的是局部变量，因此不会影响原始参数的值；而 UpdateReference 函数修改的是引用对象的成员，因此会影响原始参数。

总之，在 C# 中，函数参数的传递方式取决于参数的类型，即值类型按值传递，引用类型按引用传递。
